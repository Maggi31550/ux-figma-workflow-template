# Auto-Save Draft Is Non-Negotiable for Multi-Step Compliance Forms

**Category:** Pattern

**Source:** OCPB DataSure research, 2026-05-12

**2-Year Check:** Session interruption is a permanent feature of mobile-first environments — phone calls, network drops, battery death. Compliance forms requiring physical documents will always span multiple sessions. This is a structural truth independent of 5G rollout or device improvements.

Users completing multi-step forms that require physical documents (licences, certificates, ID numbers) routinely cannot complete them in a single session. Without auto-save and an explicit "save and return later" action, any interruption forces a full restart — a friction so high it causes permanent abandonment. Auto-save should trigger every 30 seconds and on every step transition, and a persistent "resume draft" entry point must be visible on the authenticated dashboard.

**Applies to:** Multi-step compliance forms, permit applications, e-government onboarding, insurance applications, professional registration portals — any flow where the user must gather information from offline sources.
