# Trust Signal Must Land Within 3 Seconds

**Category:** Behavior

**Source:** OCPB DataSure research, 2026-05-12

**2-Year Check:** Nielsen's response-time thresholds (1, 10 seconds) predate smartphones and have only been reinforced by mobile usage data. Consumer patience for verification interactions has not grown — if anything, TikTok-era attention spans have shortened it further. This threshold will remain valid through at least 2028.

When a consumer scans a QR code on a product, they make a trust decision within the first 3 seconds of the landing page loading. If the primary status signal (approved / not found / revoked) is not visually dominant on first viewport — or if the page is still loading — the user abandons and does not return. The trust signal must combine icon, colour, and text simultaneously (never colour alone) to pass WCAG 1.4.1 and to communicate status to colour-blind and low-vision users.

**Applies to:** Any QR-based verification system, product authentication pages, government e-service scan results, certificate verification landing pages.
